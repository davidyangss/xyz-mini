# /xyz/init 维护说明

## 目标

`/xyz/init` 用来描述 `/xyz` 工作区的结构、规则和关键演进信息，让新接手的人或 AI 能快速理解并复现环境。

## 编写原则

- 以目录为单位记录信息。
- 文档路径尽量镜像真实目录路径。
- 每个重要目录都应有对应的 `init.md`。
- 涉及安装、挂载、初始化时，写出可执行步骤。
- 暂时没有完整信息时，明确标记“待补充”。

## 一级目录规划

- `/xyz/init`：文档与初始化知识库。
- `/xyz/home`：用户目录、挂载路径、个人环境相关内容。公开版不提交真实用户文件。
- `/xyz/opt`：额外安装的软件、工具链、SDK 或第三方依赖。
- `/xyz/share`：共享资料、模板、公共资源。
- `/xyz/tools`：自建工具和辅助脚本。
- `/xyz/var`：缓存、状态文件、运行输出或日志。公开版不提交真实数据。
- `/xyz/etc`：配置模板、配置片段和环境配置说明。
- `/xyz/bin`：直接可执行的命令入口和包装脚本。

## SDK 范围

公开版只维护以下 SDK 文档：

- Go
- Rust
- Java
- Python
- Node.js

其他工具请作为私有扩展或独立插件维护，不放入公开模板。

## 维护要求

- 新增重要目录时，同步新增对应的 `init.md`。
- 目录职责变化时，同步更新这里的说明。
- 安装软件时，同步更新 SDK 文档、环境脚本和 profile.d 索引。
- 文档服务于接手、排障和环境重建，不追求堆砌。
